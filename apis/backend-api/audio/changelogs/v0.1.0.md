# Audio Backend API Changelog

## v0.1.0 (2026-06-14)

Initial release of the SDKWork Audio Backend API.

### New Features

#### Provider Route Management
- `GET /backend/v3/api/audio/provider-routes` - List provider routes
- `POST /backend/v3/api/audio/provider-routes` - Create provider route
- `GET /backend/v3/api/audio/provider-routes/{providerRouteId}` - Get provider route
- `PATCH /backend/v3/api/audio/provider-routes/{providerRouteId}` - Update provider route

#### Task Administration
- `GET /backend/v3/api/audio/tasks` - List all tasks (admin)
- `GET /backend/v3/api/audio/tasks/{taskId}` - Get task details (admin)
- `POST /backend/v3/api/audio/tasks/{taskId}/retry` - Retry failed task
- `POST /backend/v3/api/audio/tasks/{taskId}/reconcile` - Reconcile stuck task

#### Webhook Management
- `POST /backend/v3/api/audio/provider-webhooks/{providerCode}` - Accept provider webhook
- `GET /backend/v3/api/audio/webhook-events` - List webhook events
- `POST /backend/v3/api/audio/webhook-events/{eventId}/replay` - Replay webhook event

#### Analytics
- `GET /backend/v3/api/audio/analytics/usage` - Get usage analytics

### Security

- All endpoints require dual-token authentication (Authorization + Access-Token)
- Webhook endpoints use API key authentication
- Rate limiting applied per tenant

### Error Handling

- RFC 7807 Problem Details format for error responses
- Validation errors (400)
- Unauthorized errors (401)
- Rate limit errors (429)

### Pagination

- Cursor-based pagination for list operations
- Configurable page size (default: 20, max: 100)
